Functional requirements

  • The system must keep the registration information in a secure database.
  • The system must enable users to sign in with their Google accounts.
  • The system must store passwords as hashes to ensure a higher level of security.
  • If an identification or password error occurs, the system should notify the user with a message.
  • Each time the user attempts to login, the system must authenticate the identifiers to allow access..
  • The system must provide a login/registration interface to allow users to have their own ID to operate the application.
  • The system must allow users to reset their password by clicking on "I forgot my password" and receiving a link to their verified email address.
  • The system should provide an interface to address related questions to the user..
  • The system must integrate the state-of-the-art algorithm into the app to provide answers to the user queries.
  • The training dataset should be updated at regular intervals to improve the Ai model.
  • The system must provide a history delete button and authorize the deletion of the previous history of queries to the user.
  • The system should update the stored panel code and constitution based on the changes..
  • The system needs to provide a share button to allow users to share results through msg using a link.
  • The system must automatically log off when not in use for 30 minutes.
  • The system must ensure complete confidentiality by enforcing end-to-end encryption on query entries.
  • The system should display the time necessary for the system to process the request and generate the result on a screen.
  • The system should maintain a dashboard for each user containing the history of all requests placed their respective answers.
  • The system must allow the user to submit feedback through a contact form in the app.
  • The system should provide a print button so that the user can download the response provided by the Ai in PDF format.
  • The system should provide a multithreaded approach to manage multiple active users in parallel.
